# Package Rename: what_am_i_doing → waid

**Date:** 2026-05-02  
**Status:** Approved  

## Problem

The project uses `waid` as its CLI name, D-Bus namespace, and user-facing brand. But the Python package is `what_am_i_doing` — a verbose, inconsistent name that creates a split between how users talk about the tool and how developers import it. The nested `src/what_am_i_doing/` directory also confuses new contributors.

## Goals

1. Rename Python package from `what_am_i_doing` to `waid`.
2. Update all imports, references, and packaging configuration to match.
3. Preserve all existing behavior — this is a pure rename with no functional changes.
4. Make the codebase consistent: CLI name = package name = import name.

## Design

### Directory rename

```
src/what_am_i_doing/  →  src/waid/
```

The `src/` layout is preserved (standard Python src-layout pattern), just the package directory changes name.

### Packaging (pyproject.toml)

```toml
[project.scripts]
waid = "waid.__main__:main"

[tool.setuptools.package-data]
waid = ["resources/gnome/*", "resources/systemd/*"]
```

### Source files (src/waid/)

- Relative imports (`from .`) are unchanged.
- `resources.py`: `files("what_am_i_doing")` → `files("waid")`
- `service.py`: `"-m", "what_am_i_doing"` → `"-m", "waid"`

### Test files (tests/)

- All `from what_am_i_doing` → `from waid`
- All `patch("what_am_i_doing.*")` → `patch("waid.*")`
- `test_extension_assets.py`: `src/what_am_i_doing/resources/` → `src/waid/resources/`
- `test_service.py`: `"-m what_am_i_doing"` → `"-m waid"`

### Standalone scripts

- `sp-generate-tasks.py`: `from what_am_i_doing` → `from waid`

### Extension metadata

- `metadata.json` (both copies): URL field stays as `what-am-i-doing` (it's the repo name, not the package name).

### Documentation and config

- `AGENTS.md`: Update package name convention and directory references.
- `README.md`: Update any import examples.
- `skills/waid-train/SKILL.md`: Update path references.

### Cleanup

- Delete `src/waid.egg-info/` (old egg-info referencing `what_am_i_doing`); it will be regenerated by `pip install -e .`.
- Delete `src/what_am_i_doing/` directory after moving all contents to `src/waid/`.

## Verification

1. `uv sync` or `pip install -e .` — package installs as `waid`.
2. `uv run waid --help` — CLI works.
3. `uv run python -m unittest discover -s tests -v` — all tests pass.
4. `uv run python -c "from waid.constants import APP_NAME; print(APP_NAME)"` — prints `waid`.
5. No remaining references to `what_am_i_doing` in source code (except metadata URL).
